# DOCX READER

DOCX READER is a lightweight desktop application for reading **and editing** Microsoft Word `.docx` files in a clean interface.
It runs as a standalone Electron app on top of the same Express + vanilla JS architecture.

## What It Does

- Upload a `.docx` file via drag-and-drop or file picker
- Validate file type and upload size
- Convert `.docx` to HTML using Mammoth
- Sanitize output HTML before rendering
- Edit the document inline with a rich-text toolbar (bold, italic, underline, strike, headings, lists, blockquote, code block, horizontal rule, undo/redo)
- Export the edited document back to `.docx` via the Download button
- Clear/reset to upload another file

## Tech Stack

- Electron (desktop shell)
- Node.js + Express (internal server)
- Mammoth (DOCX → HTML)
- html-to-docx (HTML → DOCX on export)
- sanitize-html (input + export sanitation)
- Multer (memory upload middleware)
- Tiptap v3 + ProseMirror (editor)
- esbuild (one-shot client bundle)
- Vanilla JavaScript (ES modules)
- HTML + CSS

## Why Mammoth

Mammoth converts `.docx` into clean, semantic HTML suited for browser reading and editing.
`parseDocxToHtml` is a renderer abstraction so you can swap in `docx-preview` later if higher visual fidelity is needed.

## Install

```bash
npm install
npm run build:client
```

`build:client` bundles Tiptap from `client-src/tiptap-entry.js` into `public/js/vendor/tiptap.bundle.js`. This is required before the first run and after upgrading Tiptap deps.

## Run

Run as desktop app (recommended):

```bash
npm start
```

Alternative command:

```bash
npm run desktop
```

Run the web server only:

```bash
npm run web
```

Run web server with watch mode:

```bash
npm run web:dev
```

Rebuild the client bundle with source maps for debugging:

```bash
npm run build:client:dev
```

## Standalone Architecture

- Electron main process starts the internal Express app on an ephemeral `127.0.0.1` port.
- `BrowserWindow` loads that local URL.
- Upload, edit, and export flow runs entirely inside the local Express app.
- External links are opened in the system browser instead of navigating away from the app.

## API Endpoints (Internal)

### `POST /api/upload-docx`

Accepts multipart upload with field name `file`.

Success response:

```json
{
  "success": true,
  "fileName": "example.docx",
  "fileSize": 14325,
  "fileSizeLabel": "14 KB",
  "html": "<h1>Example</h1>",
  "stats": {
    "wordCount": 250,
    "characterCount": 1420
  },
  "warnings": []
}
```

Error response:

```json
{
  "success": false,
  "error": "Invalid file type"
}
```

### `POST /api/export-docx`

Accepts JSON body with the edited HTML and original file name.

Request:

```json
{
  "html": "<p>edited content</p>",
  "fileName": "example.docx"
}
```

Success response: a binary `.docx` file (`Content-Disposition: attachment; filename="example-edited.docx"`).

Error response:

```json
{
  "success": false,
  "error": "Unable to build .docx from the edited content."
}
```

## Editor Notes

- The toolbar provides block and inline formatting plus undo/redo.
- Document state is tracked as **Saved** or **Modified** in the header meta panel.
- `Ctrl+B`, `Ctrl+I`, `Ctrl+U`, `Ctrl+Z`, `Ctrl+Y` work natively via Tiptap.
- On DOCX export, edited HTML is sanitized again before conversion.
- If Tiptap's HTML parser fails on unusual content (rare: long paragraph-heavy documents), the client falls back to a plain-text JSON doc so the content still loads — inline formatting is dropped in that fallback.

## Security and Safety Notes

- Accepts only `.docx` extension
- Validates MIME type and max upload size
- Uses in-memory uploads (no arbitrary file writes)
- Sanitizes HTML on both import (server render) and export (DOCX build)
- Returns friendly error messages without server internals
- Electron window blocks external navigation and opens external links in system browser

## Known Limitations

- Rendering may not perfectly match Microsoft Word layout
- Tables, anchors, and raw inline styles are stripped during import for editor compatibility
- Complex formatting and advanced features may render imperfectly
- Intended for readable display and light editing, not pixel-perfect Word fidelity
- Legacy `.doc` files are not supported
